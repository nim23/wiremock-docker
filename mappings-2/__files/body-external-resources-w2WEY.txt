      <h1 id="external-resources">External Resources</h1>

<p>Code, articles and videos related to WireMock from around the web.</p>

<h2 id="integrations">Integrations</h2>
<p>Mark Winteringham wrote a very handy Chrome extension to provide a UI over WireMock:<br />
<a href="http://www.mwtestconsultancy.co.uk/wiremock-chrome-extension/">http://www.mwtestconsultancy.co.uk/wiremock-chrome-extension/</a></p>

<p>Spring Contract Verifier (previously called Accurest) is a consumer driven contracts tool that generates WireMock stub mappings as
examples for client testing.
<a href="http://cloud.spring.io/spring-cloud-contract/">http://cloud.spring.io/spring-cloud-contract/</a></p>

<p>A Spring REST Docs integration for WireMock that generates WireMock stub mappings from your test cases:<br />
<a href="https://github.com/epages-de/restdocs-wiremock">https://github.com/epages-de/restdocs-wiremock</a></p>

<h2 id="extensions">Extensions</h2>
<p>Simulate webhooks with this extension:<br />
<a href="https://github.com/wiremock/wiremock-webhooks-extension">https://github.com/wiremock/wiremock-webhooks-extension</a></p>

<p>Some folks at Open Table have written a response transformer for injecting data from the
request body into the response:<br />
<a href="https://github.com/opentable/wiremock-body-transformer">https://github.com/opentable/wiremock-body-transformer</a></p>

<p>In a similar vein, Adam York has written a response transformer utilising Velocity templates:<br />
<a href="https://github.com/adamyork/wiremock-velocity-transformer">https://github.com/adamyork/wiremock-velocity-transformer</a></p>

<p>Mason Malone has built an extension for matching requests based on the contents of JSON web tokens:<br />
<a href="https://github.com/MasonM/wiremock-jwt-extension">https://github.com/MasonM/wiremock-jwt-extension</a></p>

<p>Also from Mason, an extension for finding and removing unused stub mappings:<br />
<a href="https://github.com/MasonM/wiremock-unused-stubs-extension">https://github.com/MasonM/wiremock-unused-stubs-extension</a></p>

<h2 id="other-languages">Other languages</h2>

<p>PHP client by Rowan Hill:<br />
<a href="https://github.com/rowanhill/wiremock-php">https://github.com/rowanhill/wiremock-php</a></p>

<p>Ruby wrapper by Jeffres S. Morgan:<br />
<a href="https://rubygems.org/gems/service_mock">https://rubygems.org/gems/service_mock</a></p>

<p>Groovy binding by Tom Jankes:<br />
<a href="https://github.com/tomjankes/wiremock-groovy">https://github.com/tomjankes/wiremock-groovy</a></p>

<p>Python client by Cody Lee:<br />
<a href="https://pypi.python.org/pypi/wiremock/1.1.1">https://pypi.python.org/pypi/wiremock/1.1.1</a></p>

<h2 id="articles">Articles</h2>

<p>Sam Edwards has been hugely helpful in getting WireMock onto the Android platform and helping others do so. Here is his blog post explaining
how to write an Espresso test using WireMock as your app’s back-end:<br />
<a href="http://handstandsam.com/2016/01/30/running-wiremock-on-android/">http://handstandsam.com/2016/01/30/running-wiremock-on-android/</a></p>

<p>Dusan Dević at Yenlo wrote a useful guide to testing error conditions in the WSO2 ESB using Wiremock:<br />
<a href="https://www.yenlo.com/blog/wso2torial-error-handling-in-wso2-esb-with-wiremock">https://www.yenlo.com/blog/wso2torial-error-handling-in-wso2-esb-with-wiremock</a></p>

<p>Phill Barber has written a couple of interesting posts about practical testing scenarios with WireMock:<br />
<a href="http://phillbarber.blogspot.co.uk/2015/05/how-to-write-end-to-end-tests-for-nginx.html">http://phillbarber.blogspot.co.uk/2015/05/how-to-write-end-to-end-tests-for-nginx.html</a><br />
<a href="http://phillbarber.blogspot.co.uk/2015/02/how-to-test-for-connection-leaks.html">http://phillbarber.blogspot.co.uk/2015/02/how-to-test-for-connection-leaks.html</a></p>

<p>Bas Dijkstra kindly open sourced the content for the workshop he ran on WireMock and REST Assured:<br />
<a href="http://www.ontestautomation.com/open-sourcing-my-workshop-on-wiremock/">http://www.ontestautomation.com/open-sourcing-my-workshop-on-wiremock/</a></p>

<h2 id="videos">Videos</h2>

<p>Fluent and thorough live-coding demonstration of WireMock delivered by Sam Edwards at DevFest DC:<br />
<a href="https://youtu.be/x3MvZ8DFrpE">https://youtu.be/x3MvZ8DFrpE</a></p>

<p>Sebastian Daschner presents a step-by-step guide to running your acceptance tests in Kubernetes using WireMock:<br />
<a href="https://blog.sebastian-daschner.com/entries/acceptance_tests_wiremock_kubernetes">https://blog.sebastian-daschner.com/entries/acceptance_tests_wiremock_kubernetes</a></p>

<p>Interesting and detailed presentation by Lotte Johansen on testing microservices with WireMock at Norway’s top online marketplace:<br />
<a href="https://www.youtube.com/watch?v=cmJfMnGK-r0">https://www.youtube.com/watch?v=cmJfMnGK-r0</a></p>

<p>Chris Batey did an excellent talk at Skillsmatter in London about building fault tolerant microservices. He showed some practical
failure testing strategies using WireMock and Saboteur he’d used for real while working at Sky:<br />
<a href="https://skillsmatter.com/skillscasts/5810-building-fault-tolerant-microservices">https://skillsmatter.com/skillscasts/5810-building-fault-tolerant-microservices</a></p>

<p>Daniel Bryant’s excellent QCon presentation “The Seven Deadly Sins of Microservices” covers the full gamut of microservice anti-patterns seen in the wild, with some sound advice on how to fix them. WireMock and Saboteur get an honourable mention in the testing discussion:<br />
<a href="https://www.infoq.com/presentations/7-sins-microservices">https://www.infoq.com/presentations/7-sins-microservices</a></p>

<p>The folks at Intuit have built a very impressive and ambitious testing setup, using WireMock to isolate individual services. Here’s a talk they did at AWS:Reinvent:<br />
<a href="https://www.youtube.com/watch?list=PLhr1KZpdzuke5pqzTvI2ZxwP8-NwLACuU&amp;v=sUsh3EnzKKk">https://www.youtube.com/watch?list=PLhr1KZpdzuke5pqzTvI2ZxwP8-NwLACuU&amp;v=sUsh3EnzKKk</a></p>

<p>Michael Bailey was the first person to publicly demonstrate the possibility of running WireMock on Android. Here’s his presentation at Google’s GTAC conference on
the testing setup used by his team at Amex:<br />
<a href="https://www.youtube.com/watch?v=-xQCNf_5NNM">https://www.youtube.com/watch?v=-xQCNf_5NNM</a></p>

<p>Tom and Rob Elliot gave a join talk at Skillsmatter about patterns for readable and scalable tests with WireMock, and an approach for unit testing a
CDN:<br />
<a href="https://skillsmatter.com/skillscasts/6853-scalable-management-of-test-data-making-tests-readable">https://skillsmatter.com/skillscasts/6853-scalable-management-of-test-data-making-tests-readable</a></p>
